# Roadmap

## Phase 1: Credible Core

- Solidify mission planning primitives.
- Expand safety policy coverage.
- Add structured workspace indexing.
- Add richer test fixtures.

## Phase 2: Team Runtime

- GitHub issue and PR orchestration.
- Slack and Telegram notification adapters.
- Review queues and approval workflows.
- Shared memory profiles per repo or team.

## Phase 3: Operator OS

- Live TUI mission dashboard.
- Long-running background automations.
- Agent role specialization.
- Evaluation and replay tooling.

## Phase 4: Public Platform

- Plugin SDK.
- Hosted telemetry and observability options.
- Distribution packages.
- Demo site and launch assets.
